## Gotchas

- **min10y vs nasdaq100 naming**: dodo.py's `pull:transcripts` and `clean:transcripts` produce `nasdaq100_*` files (the broader Nasdaq-100, not the 91-ticker min10y subset). The on-disk processed parquets that downstream tasks read (`embed_transcripts`, `score_transcript_lm`, etc.) are the `nasdaq100_*` ones — not the `_min10y_coverage` variants. The 91-ticker filter happens implicitly downstream via signal-coverage joins, not at extraction. The `_min10y_coverage` meta CSVs exist in `_data/transcripts/_meta/` but **no current script produces them** — they were generated by hand or by an earlier script version and frozen.
- **`build_nasdaq100_universe.py`** does a Wikipedia fetch — no WRDS needed. `build_ciq_company_mapping.py` does need WRDS credentials.
- **Quarto kernel**: `quarto render` needs `jupytext` and `jupyter` on PATH. The dodo task and any manual render must prefix `env PATH="/opt/miniconda3/envs/genai_project/bin:$PATH"`. Documented in README under Prerequisites.
- **Metrics JSON files live in `_data/`, not `_output/`** — counterintuitive given `_output/` holds the PNG charts. The write-up reads JSON from `../_data/`.
- **Hybrid Quarto exec is reproducible but kernel-dependent**: changing `_data/metrics_*.json` doesn't trigger re-render unless `doit write_report` is used (the task has them as `file_dep`).
- **Cumulative-return chart sizes**: bumped from 10×4 / 10×5 to 12×5 / 12×5.5 for write-up legibility. If they look squashed in any other consumer, that's why.
- **`gen ai project.md` (filename has a space)** is the authoritative scope doc, linked from README via URL-encoding (`gen%20ai%20project.md`). Don't rename without updating those links.
